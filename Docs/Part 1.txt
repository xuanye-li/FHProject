1.1 Getting to Know Browser ExtensionsAdd commentMore actions
Extension: changes how the browser behaves, looks, or interacts with websites, like an app for your browser.

Structure:
my-extension/
├── manifest.json          # 🧭 The config file that defines the extension
├── background.js          # ⚙️ Runs in the background (via service worker)
├── content.js             # 📄 Injected into web pages (DOM manipulation)
├── popup.html             # 🖼 The popup UI (opens when you click the extension icon)
├── popup.js               # 🎮 Logic for the popup
├── styles.css             # 🎨 Styles for popup or content scripts
├── icon.png               # 🖼 Toolbar or store icon
└── assets/                # 📁 Optional folder for images/fonts/etc.

    }
  ],

    "48": "icon.png"
  }
}


1.2 Capabilities of Browser Extensions
API Summary
chrome.accessibilityFeatures – Manage Chrome's accessibility features (e.g., font scaling, captions).
chrome.action – Control the toolbar icon (badge, popup, title) for Manifest V3 (Chrome 88+).
chrome.alarms – Schedule your background tasks periodically or at a specific time. 
chrome.bookmarks – Create, categorize, and manage browser bookmarks. 
chrome.browsingData – Programmatically clear history, cache, cookies, etc. 
chrome.commands – Define keyboard shortcuts in manifest. 
chrome.contentSettings – Control site permissions (cookies, JavaScript, etc.)
chrome.contextMenus – Add custom entries to right-click menus. 
chrome.cookies – Read/write cookies and listen to changes.
chrome.debugger – Use DevTools Protocol via tabs.
chrome.declarativeContent – React to page content without direct script injection.
chrome.desktopCapture – Capture screen, window, or tab content.
chrome.dom – DOM manipulation with elevated privileges.
chrome.downloads – Control downloads programmatically.
chrome.events – Standard event interfaces for extension callbacks.
chrome.extension – Utilities for messaging and view management.
chrome.extensionTypes – Type definitions used across APIs. 
chrome.fontSettings – Change browser font preferences.
chrome.gcm – Firebase Cloud Messaging support.
chrome.history – View and manipulate browser history.
chrome.i18n – Support multiple languages in UI.
chrome.identity – OAuth2 and user identity workflows. 
chrome.idle – Detect when the user or system is inactive.
chrome.instanceID – Unique ID for each extension instance.
chrome.management – Query and control installed extensions/apps. 
chrome.notifications – Display system notifications.
chrome.offscreen – Use hidden documents for DOM or WebGL work.
chrome.runtime – Lifecycle, messaging, metadata.
chrome.permissions – Request and check permissions at runtime.
chrome.scripting – Inject scripts/styles in MV3.
chrome.sidePanel – Add a side panel in Chrome’s UI. 
chrome.storage – Store user data (local, sync, session). 
chrome.tabs – Manage browser tabs (create, query, modify).
chrome.userScripts – Manage programmatic injection of scripts. 
chrome.webNavigation – Monitor navigation events in frames.
chrome.webRequest – Observe or modify HTTP requests. 
chrome.windows – Control browser windows (open, resize).
